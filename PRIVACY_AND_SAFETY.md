## 1. Data Sovereignty & Zero-Exfiltration Guarantee

In heavy industrial manufacturing, oil & gas refining, and critical energy infrastructure, Standard Operating Procedures (SOPs), Piping and Instrumentation Diagrams (P&IDs), and real-time SCADA telemetry represent highly confidential intellectual property and national critical infrastructure. Exposing these data assets to external multi-tenant cloud LLM APIs (e.g., OpenAI, Anthropic, or public cloud vector stores) creates severe cybersecurity vulnerabilities, regulatory non-compliance, and espionage risks.

**Ops Brain Local guarantees 100% data sovereignty by architecture:**
- **Zero Cloud Transmission:** All document parsing, vector generation, graph entity extraction, and LLM token generation run entirely within the physical RAM and local disk storage of the host machine.
- **Zero Telemetry:** All third-party library telemetry (including `ChromaDB PostHog analytics` and `HuggingFace Hub update checks`) is explicitly disabled via code-level flags (`local_files_only=True` and environment overrides).
- **Physical Air-Gap Compatibility:** The entire system functions seamlessly without a network interface card (NIC), or when Wi-Fi and Ethernet connections are physically severed (`Airplane Mode`).

## 2. Local Storage Persistence & Directory Boundaries

All user uploads, parsed text chunks, vector embeddings, and relational graph structures are confined strictly within the local repository directory (`data/`). Nothing is ever written to system temporary directories (`/tmp` or `C:\Windows\Temp`) or external cloud storage buckets:

| Local Directory / File Path | Data Stored | Access Permissions | Retention & Deletion Policy |
| :--- | :--- | :--- | :--- |
| **`data/uploads/`** | Raw uploaded files (`.pdf`, `.xlsx`, `.csv`, `.docx`) | Local OS user read/write | Persists until user clicks `Delete Document` in Ingestion Studio UI or clears data directory. |
| **`data/chroma/`** | `ChromaDB` embedded vector index (`384-dim dense embeddings`) | Local OS user read/write | Synchronized directly with ingested documents; fully wiped when collections are reset. |
| **`data/metadata.db`** | `SQLite 3` relational database storing document titles, asset tags, work orders, and statutory rules | Local OS user read/write | ACID-compliant local database; never backed up to cloud. |
| **`data/graph.pkl`** | `NetworkX` serialized multi-graph storing `spaCy` NER nodes and relational edges | Local OS user read/write | Updated in real-time during each document ingestion cycle. |
| **`models/qwen2.5-3b-int4/`** | Quantized OpenVINO `Qwen2.5-3B` IR model (`.xml` / `.bin` weights) | Local OS user read-only | Static local weights directory downloaded once during setup (`scripts/download_model.py`). |

## 3. Industrial Hallucination Prevention & Safety Guardrails

In an industrial plant, if an AI assistant hallucinates a pump discharge pressure limit (*e.g., generating `85.0 Bar` instead of the true `42.5 Bar` rating*) or omits a required lockout/tagout (LOTO) step, the consequences can include mechanical overpressure, chemical explosions, or fatal injury.

To eliminate generative hallucinations, **Ops Brain Local** implements a **Deterministic RAG Safety Shield (`core/pipeline.py`)** with four mandatory guardrails:

### Guardrail 1: Deterministic Confidence Badging (`HIGH` / `MEDIUM` / `LOW`)
Every response generated by the Co-Pilot (`/copilot`) undergoes real-time cosine similarity thresholding against retrieved vector chunks:
- **`HIGH` Confidence (Green Badge - Cosine Score > 0.65):** The generated answer is directly supported by high-similarity chunks from ingested manuals.
- **`MEDIUM` Confidence (Yellow Badge - Cosine Score 0.35 to 0.65):** Partial documentation retrieved; the AI explicitly advises verifying specific parameters with field supervisors.
- **`LOW` Confidence (Red Badge - Cosine Score < 0.35):** The query asks for information not present in the local database. The system outputs a mandatory warning: *"WARNING: No exact operating manual or SOP chunk found in local storage matching this query. Do NOT execute field maintenance without consulting the equipment manufacturer or plant engineering lead."*

### Guardrail 2: Interactive Citation Accordions (Auditable Paper Trail)
Every AI answer includes expandable **Citation Accordions** detailing the exact source file (`SOP_P204_Crude_Pump_Maintenance_Manual.pdf`), chunk ID (`DOC-C980D8CE_CHUNK_2`), and exact text excerpt. Field technicians can instantly verify the AI's claim against the verbatim source document before touching a physical valve.

### Guardrail 3: Tickable Action Checklists (`[DONE]` Verification)
To prevent maintenance technicians from accidentally skipping mandatory preparation steps (*e.g., skipping atmospheric gas testing before Confined Space Entry*), our backend parses all procedural recommendations and renders them as **Interactive Tickable Checklists** in the UI. Each step can be checked off (`[x]`) as completed, creating a clear, step-by-step verification workflow.

### Guardrail 4: Statutory & Regulatory Compliance Cross-Checking (`/compliance`)
The dedicated **Statutory Compliance Audit Studio (`/compliance`)** cross-references active plant SOPs against local and international safety frameworks:
- **`OISD-GDN-192 Section 4.2`** (Oil Industry Safety Directorate guidelines for Confined Space Entry and Hot Work permits).
- **`Factory Act 1948`** (Statutory occupational safety and machinery guarding regulations).
- **`API 610 / API 682`** (American Petroleum Institute standards for centrifugal pumps and mechanical seal flushes).
If an uploaded maintenance manual lacks mandatory statutory provisions (*e.g., missing a required H2S gas check threshold*), the system marks the asset as `PARTIAL` or `NON_COMPLIANT` in the audit matrix, alerting reliability leadership immediately.
